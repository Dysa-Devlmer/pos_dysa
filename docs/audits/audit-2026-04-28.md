## 3 · Incidentes recientes (2026-04-26 → 2026-04-27)

### 🔥 Incidente prod 2026-04-27 — schema desync (CERRADO)

**Síntoma:** `/` devuelve 500 con digest 3553871261 en prod. Logs Sentry: `P2022 column does not exist` para `ventas.deleted_at` y tablas F-9 (`cajas`, `aperturas_caja`, `movimientos_caja`, `pagos_venta`).

**Causa raíz:** el container `pos-web` corre código F-3+F-9 nuevo pero la BD se creó con `db push` al inicio y nunca se aplicó migrations en deploys posteriores. El Dockerfile multi-stage **NO copia `packages/db/prisma/`** al final stage → imposible correr `prisma migrate deploy` desde el container.

**Fix definitivo (commit `d823990`, gotcha 96):** nueva fase 5b/6 en `scripts/deploy.sh`:
1. `tar -czf prisma-migrations.tar.gz packages/db/prisma/` localmente.
2. `scp` al VPS `/tmp/`.
3. `docker run --rm --network pos-chile-network node:22-alpine sh -c "npx -y prisma@6.x migrate deploy"` ad-hoc.
4. Cleanup `/tmp`.

**Mitigación:** las migraciones son **idempotentes** (`IF NOT EXISTS`, `DO $$ EXCEPTION`) — seguro re-correrlas en cada deploy.

### 🐛 React #418 hydration mismatch (CERRADO 2026-04-27 commit `1ae8226`)

- `formatFechaHora` sin `timeZone` explícito → server (UTC) vs browser (TZ local) generan strings distintos. Fix: forzar `timeZone: "America/Santiago"` (gotcha 97).
- `Sparkline` con `<ResponsiveContainer>` SSR width(-1) → mount gate `useEffect` (gotcha 98). Mismo pattern que `VentasChart` ya tenía.

### 🐛 Tab refocus jank UX (CERRADO 2026-04-27 commit `24167c9`)

- `backdrop-filter: blur` en surfaces always-on (header, data-table sticky, ventas-chart tooltip) tardaba 1-2 frames extra al salir de tab-freeze (Chromium Memory Saver, Safari background freeze, Firefox throttle).
- Fix: bg sólido en lugar de blur. Mantiene blur on-demand (mobile drawer overlay).

## E · Code Review mobile (skill `/code-reviewer`)

### 🔴 Críticos (3)

**MC1.** `stores/authStore.ts:55-122` — **No persist + logout no cascada**. Store NO usa `persist` middleware (solo SecureStore para token). Más grave: `handleFlushError` trata 429 como "otros 4xx inexistente" → fila se queda pending sin backoff, retry inmediato genera **tormenta de 429**. No se respeta `Retry-After`. **MC3.** `app/_layout.tsx:58-66` — **`AppState.addEventListener` registrado a module-scope**: MEMORY LEAK. Registro fuera de useEffect, en hot-reload de dev y al re-mount del layout se acumulan listeners (cada uno disparando `syncNow()`). En prod no hay hot-reload, pero el subscription nunca se `remove()`. Fast Refresh y re-evaluación de módulo lo duplican.

### 🟠 Altos (top 5)

### 🔴🔴 SS3 · `AppState.addEventListener` registrado a MODULE-SCOPE — race condition con bootstrap

`_layout.tsx:65-72`:
```ts

```

**Problemas reales en APK release:**

1. **Race condition con DB init.** El listener se registra al evaluar el módulo `_layout.tsx` (antes que cualquier `useEffect` corra). 2. **Memory leak en hot-reload dev** (no afecta release).

3. **No cleanup en unmount** — RootLayout nunca se desmonta en práctica, pero si hubiera un `_layout` parent, esto sería leak.

**Fix:** mover al `useEffect` de `RootLayoutNav` con cleanup `subscription.remove()`. Y agregar guard `useSyncStore.getState().isReady` antes de `syncNow()`. **15 min.**
